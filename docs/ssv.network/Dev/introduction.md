# Введение

## Запуск распределенного валидатора

В этом руководстве описаны шаги, необходимые для запуска распределенного валидатора через ssv.network с использованием
смарт-контрактов протокола и инструментов разработчика.

### Необходимы набор:

* Валидатор Ethereum - генерация ключа валидатора и внесение (регистрация) его в цепочку beacon могут быть выполнены с
  помощью <a href="https://github.com/ethereum/staking-deposit-cli">официального CLI</a> для внесения ставок Ethereum и
  их <a href="https://prater.launchpad.ethereum.org/en/">Launchpad</a>.

* Goerli ETH (<0,1) для покрытия транзакционных расходов на газ в тестовой сети Goerli (члены сообщества в
  нашем <a href="https://discord.gg/D5kUdV93">discord</a>
  могут помочь в получении необходимой суммы).

* Тестовый токен SSV(<a href="https://faucet.ssv.network/">faucet</a>)

### Обзор процесса

Запуск распределенного валидатора состоит из 3 этапов:

1. Выберите группу операторов для управления вашим валидатором.
2. Распространите свой ключ валидатора среди участников обмена ключами.
3. Зарегистрируйте свой валидатор в сети.

![Alt text](/img/ssv/ssv_developer_001.png)

#### 1. Выбор оператора

Выберите предпочитаемых вами операторов из реестра операторов наших сетей - для каждого валидатора требуется 4
оператора.

Для каждого выбранного оператора вы должны получить его идентификатор, присвоенный сети, и соответствующий ключ.

Весь реестр операторов можно просмотреть через ssv.network explorer или через SSV API, чтобы получить доступ к
необходимым идентификаторам / ключам операторов.

![Alt text](/img/ssv/ssv_developer_002.png)

:::info

Пожалуйста, обратите внимание, что вы всегда можете настроить и обновить оператороы управляемых вашим валидатом

:::

#### 2. Распределение ключей

Чтобы назначить операцию проверки выбранным вами операторам, вы должны распространить свой ключ проверки на KeyShares.

Используйте инструменты <a href="https://wiki.chainops.org/docs/ssv.network/Dev/Tools/KeyDistributor/">распространителя
ключей SSV</a>, чтобы извлечь ваш ключ проверки подлинности из вашего файла хранилища ключей и распространить их по
разделам ключей.

#### 3. Регистрация в сети

Чтобы подать сигнал вашей операторской группе о начале работы вашего валидатора, вы должны зарегистрировать свой
валидатор в сети, передав транзакцию
в <a href="https://docs.ssv.network/developers/smart-contracts/validators-methods#registervalidator">смарт-контракт</a>
ssv.network.

Вы можете создать транзакцию самостоятельно или с помощью полезной нагрузки, сгенерированной
инструментом <a href="https://wiki.chainops.org/docs/ssv.network/Dev/Tools/KeyDistributor/">распространения ключей
SSV</a>, использованным на предыдущем шаге.

:::info

Пожалуйста, обратите внимание, что, поскольку SSV вносится в контракт, вы должны сначала утвердить адрес контракта SSV,
чтобы потратить свой токен SSV до транзакции регистрации.

:::

В качестве альтернативы вы можете использовать интерфейс <a href="https://app.prater.ssv.network/">webapp</a> для создания и управления вашим валидатором.
